# Git & GitHub Fundamentals

This repository serves as a practical guide to basic version control concepts and GitHub workflow.

## 1. What is Version Control?
Version Control is a system that records changes to a file or set of files over time so that you can recall specific versions later. It allows multiple developers to work on the same project simultaneously.

## 2. Git vs. GitHub
* Git: A local version control tool used to manage source code history.
* GitHub: A cloud-based hosting service for Git repositories.

## 3. GitHub Alternatives
1. GitLab
2. Bitbucket
3. SourceForge

## 4. Git Fetch vs. Git Pull
* Git Fetch: Downloads the latest changes but does not merge them.
* Git Pull: Downloads changes and automatically merges them into your current branch.

## 5. Git Rebase
Rebase is the process of moving or combining a sequence of commits to a new base commit to create a linear history.
* Command: git rebase <branch-name>

## 6. Git Cherry-pick
Cherry-pick allows you to pick a specific commit from one branch and apply it to another.
* Command: git cherry-pick <commit-hash>
